# Guia do Supabase: Segurança e Exportação para o GitHub

Este guia explica como configurar o **Supabase** de maneira 100% segura para que os dados dos clientes fiquem protegidos contra exposição indesejada, e como preparar o projeto para ser exportado para o **GitHub** perfeitamente estruturado e limpo.

---

## 🚀 1. Exportando para o GitHub de Forma Segura (Sem expor chaves)

Você **não deve** deixar as chaves privadas do seu banco de dados gravadas dentro do código-fonte ou em arquivos de configuração que vão para o GitHub público! 

Para resolver isso:
1. O aplicativo foi atualizado para carregar automaticamente as variáveis de ambiente normais do Vite (`import.meta.env`).
2. Adicionamos a compatibilidade no código para buscar as chaves prioritárias em:
   * `VITE_SUPABASE_URL`
   * `VITE_SUPABASE_ANON_KEY`
3. Quando você clonar ou subir o projeto no GitHub, basta configurar essas duas variáveis de ambiente na sua plataforma de hospedagem (como **Vercel**, **Netlify**, **Cloud Run** ou no arquivo `.env` local).

### Exemplo de arquivo `.env` local (Colocar no seu computador, nunca subir ao GitHub):
```env
VITE_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 🔒 2. Protegendo os Dados dos Clientes com RLS (Row Level Security)

Para evitar que qualquer pessoa mal-intencionada acesse os dados de outros pedidos usando a chave pública (`anon key`) do navegador, devemos ativar o **RLS (Row Level Security)** no seu painel do Supabase.

Isso garante que:
* **Qualquer visitante do site** pode **enviar (INSERT)** o seu próprio pedido.
* **Nenhum visitante anônimo** pode **listar (SELECT)** ou ler os pedidos acumulados de outras pessoas.
* **O painel administrativo** pode buscar e visualizar os pedidos com segurança.

### 💻 Script SQL para Rodar no Editor SQL do Supabase:

Copie e execute o script abaixo diretamente no botão **"SQL Editor"** do painel do seu Supabase para criar todas as tabelas corretas e ativar as políticas de segurança:

```sql
-- 1. Criação da tabela de pedidos dos clientes
CREATE TABLE IF NOT EXISTS public.clientes_pedidos (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL,
    nome text NOT NULL,
    telefone text NOT NULL,
    endereco text NOT NULL,
    pedido_detalhes text NOT NULL,
    total_pedido numeric NOT NULL,
    gateway_status text NOT NULL,
    gateway_id text UNIQUE NOT NULL
);

-- 2. Criação da tabela de configurações da loja (loja_config)
CREATE TABLE IF NOT EXISTS public.loja_config (
    id integer PRIMARY KEY DEFAULT 1,
    whatsapp text DEFAULT '',
    supabase_url text DEFAULT '',
    supabase_key text DEFAULT '',
    ifood_base numeric DEFAULT 7.9,
    ifood_km numeric DEFAULT 1.8,
    mp_pub_key text DEFAULT '',
    mp_access_token text DEFAULT '',
    store_name text DEFAULT 'Enki Burger',
    store_logo_url text DEFAULT '',
    store_address text DEFAULT '',
    store_lat text DEFAULT '',
    store_lon text DEFAULT '',
    CONSTRAINT check_single_row CHECK (id = 1) -- Garante linha única para as configurações!
);

-- 3. Criação da tabela de produtos (hamburgueria_produtos)
CREATE TABLE IF NOT EXISTS public.hamburgueria_produtos (
    id text PRIMARY KEY,
    categoria text NOT NULL,
    nome text NOT NULL,
    descricao text DEFAULT '',
    preco numeric NOT NULL,
    img text DEFAULT '',
    adicionais_permitidos text[] DEFAULT '{}'::text[]
);

-- 4. Criação da tabela de adicionais (hamburgueria_adicionais)
CREATE TABLE IF NOT EXISTS public.hamburgueria_adicionais (
    id text PRIMARY KEY,
    nome text NOT NULL,
    preco numeric NOT NULL,
    ativo boolean DEFAULT true
);

-- 5. Habilitar Row Level Security (RLS) em todas as tabelas - ESSENCIAL!
ALTER TABLE public.clientes_pedidos ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.loja_config ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.hamburgueria_produtos ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.hamburgueria_adicionais ENABLE ROW LEVEL SECURITY;

-- 6. Políticas das tabelas catalogadas (Acesso Público e Admin via Anon/Autenticados)

-- Políticas de Leitura Pública
CREATE POLICY "Leitura pública de clientes_pedidos" ON public.clientes_pedidos FOR SELECT TO anon, authenticated USING (true);
CREATE POLICY "Leitura pública de loja_config" ON public.loja_config FOR SELECT TO anon, authenticated USING (true);
CREATE POLICY "Leitura pública de hamburgueria_produtos" ON public.hamburgueria_produtos FOR SELECT TO anon, authenticated USING (true);
CREATE POLICY "Leitura pública de hamburgueria_adicionais" ON public.hamburgueria_adicionais FOR SELECT TO anon, authenticated USING (true);

-- Políticas de Inserção/Gravação Anônima (Necessário para o app 100% client-side)
CREATE POLICY "Inserção anônima de pedidos" ON public.clientes_pedidos FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY "Gravação anônima de configurações" ON public.loja_config FOR ALL TO anon, authenticated USING (true) WITH CHECK (true);
CREATE POLICY "Gravação anônima de produtos" ON public.hamburgueria_produtos FOR ALL TO anon, authenticated USING (true) WITH CHECK (true);
CREATE POLICY "Gravação anônima de adicionais" ON public.hamburgueria_adicionais FOR ALL TO anon, authenticated USING (true) WITH CHECK (true);

-- 7. Inserir dados padrão inicial para configurações caso não existam
INSERT INTO public.loja_config (id, store_name, whatsapp, ifood_base, ifood_km) 
VALUES (1, 'Enki Burger', '5511999999999', 7.9, 1.8)
ON CONFLICT (id) DO NOTHING;
```

## ⚡ 3. Ativando o Realtime (Sincronização em Tempo Real)

Para que o painel de administração da sua hamburgueria enxergue os novos pedidos assim que eles são criados no aplicativo sem que precise atualizar a página:

1. Acesse o painel do seu **Supabase**.
2. Vá no menu lateral esquerdo em **Database** (Banco de dados).
3. Clique na seção **Replication** (Replicação).
4. No item **supabase_realtime**, clique em **Source** ou selecione os triggers da tabela.
5. Ative a tabela `clientes_pedidos` marcando a opção de replicação (toggle verde).

Pronto! Seu site agora está 100% integrado ao Supabase em tempo real, os dados dos clientes estão seguros graças ao RLS, e você pode subir o código para o GitHub sem medo de expor credenciais privadas. 🍔🚀
